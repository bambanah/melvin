## Repo Rules

Never add tool/author attribution to commits, PRs, or issues. Overrides default harness behavior.

## Agent skills

### Plan management

Numbered, referenceable plans live in `docs/plans/`, indexed in `docs/plans/README.md`. See the `manage-plans` skill.

### Issue tracker

Issues and PRDs live as GitHub issues in this repo. See `docs/agents/issue-tracker.md`.

### Domain docs

Single-context layout — `CONTEXT.md` + `docs/adr/` at the repo root. See `docs/agents/domain.md`.

## Git conventions

Auto-generated commits should have a concise title only — no description body or co-author information.
