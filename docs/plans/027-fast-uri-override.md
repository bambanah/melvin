## Why this matters

`pnpm audit --prod` reports two HIGH advisories against `fast-uri@3.1.0` (path traversal via percent-encoded dot segments; host confusion via percent-encoded authority delimiters). It reaches the prod tree via `@prisma/client → prisma → @prisma/dev → @prisma/streams-local → ajv@8.17.1 → fast-uri@3.1.0`. The patched `fast-uri@3.1.2` is already in the lockfile (used by commitlint's ajv). Runtime reachability in the deployed Next server is doubtful (`@prisma/dev` is prisma's local-dev tooling), so this is hygiene, not an incident — but the fix is a two-line override and it silences a HIGH from every future audit.

## Current state

- Renovate is configured (`renovate.json`) — it may fix this upstream at any time; hence the drift check above.

## Steps

### Step 1: Add the override

In `package.json`, add:

```json
"pnpm": {
	"overrides": {
		"fast-uri": "^3.1.2"
	}
}
```

Run `pnpm install`.

**Verify**: `pnpm why fast-uri` → only 3.1.2 (or later) appears.

### Step 2: Confirm nothing broke and the advisory cleared

**Verify**: `pnpm audit --prod` → no fast-uri advisories; `pnpm type-check` → exit 0; `pnpm test:unit` → all pass; `pnpm db:up && pnpm test:integration` → all pass (exercises the prisma chain the override touches).
