## Findings considered and rejected (2026-07-07 audit — do not re-audit)

- **App Router migration**: pages router fully supported in Next 16; blast radius is every page + `@trpc/next` wiring; no TRD needs RSC. Not worth doing.
- **Auth.js v5 migration**: v4 stable; v5's wins don't apply to a pages-router solo app. Revisit only bundled with a (rejected) App Router move. TRD-001 notes an eventual auth migration — that's a future product decision, not maintenance.
- **Zod 4 migration**: perf/typing only; defer unless schema-heavy work is already underway.
- **NDIA bulk-claim CSV export**: explicitly scoped out — payer mix is plan-/self-managed (TRD assumptions, `00-ux-analysis.md`).
- **`invoice-router.ts` as god-module** (795 lines): properly layered — delegates to `lib/` modules; ~50-65 lines per procedure. Leave it.
- **Evening rate applies to whole activity keyed off `endTime`**: recorded as a deliberate, revisitable product decision in plan 004's out-of-scope notes. The weekend/weekday-gate fall-through folds into the rate engine (033 phases).
- **Trip `addActivity` leaves a bypassed inter-client leg row**: data cruft, not a billing defect (`calculateTripTransit` reads consecutive pairs only).
- **Two debounce idioms** (`generic-utils.ts` wrapper vs `invoice-list.tsx` hook): different shapes for different jobs; consolidation not worth the churn (noted in plan 028).
- **Overlap check excluding group activities** (`overlap-utils.ts:38`): defensible — mirror rows share a time slot and would self-conflict. LOW confidence it's a bug; revisit only if double-booking reports appear.
- **`user.reset` partial-delete on versioned invoices**: not rejected — _characterized_ by plan 025; fix is a follow-up product decision (reset vs. ADR-0004 version retention).
